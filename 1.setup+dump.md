# MongoDB Restore Guide from Client Dump

If your client gives you a MongoDB dump, you do not need to export from Atlas. You only need to restore the dump into your Linux MongoDB server.

## Migration Flow

```text
Client MongoDB Dump
        ↓
Upload Dump to Linux Server
        ↓
Restore into MongoDB
        ↓
Verify Data
        ↓
Update Backend Connection
```

---

## Step 1 — Copy Dump to Linux Server

From your local machine:

```bash
scp -i your-key.pem -r dump_folder ubuntu@YOUR_SERVER_IP:/home/ubuntu/
```

Example:

```bash
scp -i aws-key.pem -r mongodb_dump ubuntu@54.12.11.20:/home/ubuntu/
```

---

## Step 2 — Check Dump Folder on Server

SSH into server:

```bash
ssh -i your-key.pem ubuntu@YOUR_SERVER_IP
```

Check files:

```bash
ls /home/ubuntu/
```

You should see:

```text
mongodb_dump/
```

---

## Step 3 — Verify MongoDB Running

```bash
sudo systemctl status mongod
```

Expected:

```text
active (running)
```

---

## Step 4 — Restore Dump

If authentication is enabled:

```bash
mongorestore \
--username admin \
--password YOUR_PASSWORD \
--authenticationDatabase admin \
/home/ubuntu/mongodb_dump
```

If authentication is NOT enabled:

```bash
mongorestore /home/ubuntu/mongodb_dump
```

---

## Step 5 — Restore Specific Database (Optional)

If dump contains one DB:

```bash
mongorestore --db mydatabase /home/ubuntu/mongodb_dump/mydatabase
```

---

## Step 6 — Verify Database

Login:

```bash
mongosh
```

Check databases:

```javascript
show dbs
```

Use database:

```javascript
use mydatabase
```

Check collections:

```javascript
show collections
```

Check records:

```javascript
db.collection_name.find().limit(5)
```

---

## Step 7 — Update Backend Connection

Example:

```bash
mongodb://admin:password@SERVER_IP:27017/dbname?authSource=admin
```

---

## Step 8 — Restart Backend

For Node.js:

```bash
pm2 restart all
```

---

## Important

Ask your client which dump type they provide:

1. BSON dump (`mongodump` output) → use `mongorestore`
2. JSON export → use `mongoimport`

You can identify by checking files:

- `.bson` → `mongorestore`
- `.json` → `mongoimport`
